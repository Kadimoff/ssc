# Backend Plan — Student Startup Community

## Stack

| Layer | Seçim | Səbəb |
|-------|-------|--------|
| **Runtime** | Bun 1.2+ | TS native, built-in test runner, fast package manager |
| **Framework** | Hono 4.x | TypeScript-first, lightweight (~12KB), RPC tRPC-style |
| **Database** | PostgreSQL 16 | ACID, JSONB, array types |
| **ORM** | Drizzle ORM | Type-safe, close to SQL, migration tooling |
| **Auth** | JWT (`@hono/jwt`) | Frontend-də artıq Bearer token gözlənilir |
| **Validation** | Zod | Frontend-də də var, paylaşmaq olar |
| **Deploy** | Docker Compose | Bun + PostgreSQL — bir-command `docker compose up` |

## Database Schema (PostgreSQL)

### `users`

```sql
CREATE TABLE users (
  id            SERIAL PRIMARY KEY,
  username      VARCHAR(100) UNIQUE NOT NULL,
  password_hash TEXT NOT NULL,
  name          VARCHAR(200) NOT NULL DEFAULT '',
  email         VARCHAR(255) NOT NULL DEFAULT '',
  title         VARCHAR(200) NOT NULL DEFAULT '',
  company       VARCHAR(200) NOT NULL DEFAULT '',
  industry      VARCHAR(100) NOT NULL DEFAULT '',
  location      VARCHAR(200) NOT NULL DEFAULT '',
  skills        TEXT NOT NULL DEFAULT '',
  about         TEXT NOT NULL DEFAULT '',
  availability  VARCHAR(100) NOT NULL DEFAULT 'Open to collaborate',
  website       VARCHAR(500) NOT NULL DEFAULT '',
  role          VARCHAR(10) NOT NULL DEFAULT 'member' CHECK (role IN ('admin', 'member')),
  created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

### `posts`

```sql
CREATE TABLE posts (
  id              SERIAL PRIMARY KEY,
  author_id       INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  content         TEXT NOT NULL,
  post_type       VARCHAR(50) NOT NULL DEFAULT 'Update',
  preview_title   VARCHAR(300) NOT NULL DEFAULT '',
  preview_subtitle VARCHAR(300) NOT NULL DEFAULT '',
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

### `post_reactions`

```sql
CREATE TABLE post_reactions (
  user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  post_id INTEGER NOT NULL REFERENCES posts(id) ON DELETE CASCADE,

  PRIMARY KEY (user_id, post_id, type)
);
```

### `post_comments`

### `reposts`

### `connections`

```sql
CREATE TABLE connections (
```

## API Endpoint Map

Auth required (JWT Bearer token) — `Authorization: Bearer <token>`

| Method | Path | Auth | Admin | Request | Response | Frontend method |
|--------|------|------|-------|---------|----------|----------------|
| GET | `/snapshot` | Yes | - | - | `Snapshot` | `snapshot()` |
| POST | `/auth/login` | No | - | `{ username, password }` | `{ token, user }` | `login()` |
| POST | `/auth/register` | No | - | `Registration` | `{ token, user }` | `register()` |
| POST | `/auth/logout` | Yes | - | - | `{}` | `logout()` |
| PATCH | `/users/me` | Yes | - | `Partial<User>` | `{ user }` | `updateProfile()` |
| POST | `/posts` | Yes | - | `{ content, post_type? }` | `{}` | `createPost()` |
| POST | `/posts/:id/like` | Yes | - | - | `{}` | `togglePost(id, 'liked')` |
| POST | `/posts/:id/save` | Yes | - | - | `{}` | `togglePost(id, 'saved')` |
| POST | `/posts/:id/repost` | Yes | - | - | `{}` | `repost()` |
| POST | `/posts/:id/comments` | Yes | - | `{ text }` | `{}` | `comment()` |
| DELETE | `/admin/posts/:id` | Yes | Yes | - | `{}` | `deletePost()` |
| POST | `/connections` | Yes | - | `{ user_id }` | `{}` | `connect()` |
| POST | `/communities/:id/join` | Yes | - | `{ join: bool }` | `{}` | `toggleCommunity()` |
| DELETE | `/admin/communities/:id` | Yes | Yes | - | `{}` | `deleteCommunity()` |
| POST | `/conversations` | Yes | - | `{ user_id }` | `{ id }` | `ensureConversation()` |
| POST | `/conversations/:id/messages` | Yes | - | `{ text }` | `{}` | `sendMessage()` |
| POST | `/jobs/:id/apply` | Yes | - | - | `{}` | `toggleJob(id, 'applied')` |
| POST | `/jobs/:id/save` | Yes | - | - | `{}` | `toggleJob(id, 'saved')` |
| POST | `/admin/jobs` | Yes | Yes | `Job` (without id/applied/saved) | `{}` | `createJob()` |

### Error response format (bütün endpoint-lər üçün eyni)

```json
{
  "error": "Human-readable error message"
}
```

Status codes: `200` OK, `201` Created, `400` bad request, `401` unauthorized, `403` forbidden, `404` not found.

## Auth Flow

```
Client                          Server
  │                                │
  │  POST /auth/login              │
  │  { username, password }        │
  │  ───────────────────────────>  │  SELECT * FROM users WHERE username = ?
  │                                │  Bun.password.verify(password, hash)
  │                                │  jwt.sign({ sub: user.id, role: user.role })
  │  <───────────────────────────  │
  │  { token, user }               │
  │                                │
  │  GET /snapshot                 │
  │  Authorization: Bearer <jwt>   │
  │  ───────────────────────────>  │  @hono/jwt verify
  │                                │  c.set('userId', sub)
  │                                │  c.set('userRole', role)
```

JWT payload:
```ts
{ sub: number, role: 'admin' | 'member', iat: number, exp: number }
```

JWT expiry: 7 days (frontend localStorage-da saxlanılır, logout-da silinir).

## Seed Data

Frontend `demo-client.ts`-də olan `seedState()`-i PostgreSQL-ə import edəcəyik. Seed script-i `backend/seed.ts`:

1. `TRUNCATE` bütün cədvəllər
2. `INSERT` users (password_hash Bun.password.hash ilə)
3. `INSERT` posts, communities, jobs, conversations, messages
4. `RESET` sequence-lər (id-lər uyğun olsun deyə)

## Middleware

### `auth.ts`

```
1. Request-dən Authorization header-ını götür
2. "Bearer <token>" formatını validate et
3. @hono/jwt verify (JWT_SECRET ilə)
4. c.set('userId', payload.sub)
5. c.set('userRole', payload.role)
6. Əgər token yoxdursa və ya invalid -> 401
```

### `admin.ts`

```
1. Auth middleware-i require edir
2. c.get('userRole') === 'admin' deyilsə -> 403
```
